<h2 class="text-center text-4xl font-bold mb-8">Historia de Spring</h2>

<img class="w-full max-w-lg mx-auto rounded-md mb-10" src="/img/tech/spring.jpg" />

<div class="mb-6 space-y-6">
    <p>
        La historia de <span class="text-mb-blue-light"><strong>Spring</strong></span> y su evolución a <span class="text-mb-blue-light"><strong>Spring Boot</strong></span> es realmente increíble, sobre todo porque cambio la forma en la cual los desarrolladores Java construían aplicaciones.
        La historia comienza en <span class="text-mb-blue-light"><strong>2002</strong></span>, cuando <span class="text-mb-blue-light"><strong>Rod Johnson</strong></span> publicó su libro <span class="text-mb-blue-light"><strong>"Expert One-on-One Java EE Design and Development"</strong></span> en el que introdujo los conceptos de <span class="text-mb-blue-light"><strong>Inversión de Control (IoC)</strong></span> y la <span class="text-mb-blue-light"><strong>Inyección de Dependencias (DI)</strong></span>.
        Este libro dio origen al desarrollo del <span class="text-mb-blue-light"><strong>framework Spring</strong></span>, cuyo objetivo era simplificar la configuración de aplicaciones Java frente a la complejidad de Java EE en ese momento.
    </p>

    <div class="w-fit mx-auto">
        <img class="w-72 rounded-md" src="/img/post/rod-johnson.jpeg">
        <h2 class="text-center font-bold text-xl mt-2">Rod Johnson</h2>
    </div>

    <p>
        <span class="text-mb-blue-light"><strong>Spring Framework</strong></span> se lanzó oficialmente en <span class="text-mb-blue-light"><strong>2003</strong></span> y rápidamente se volvió popular por su capacidad de desacoplar componentes de software y simplificar el desarrollo de aplicaciones.
    </p>
</div>

<div class="mb-6 space-y-6">
    <p>
        Rod Johnson y su equipo fundaron en 2004 <span class="text-mb-blue-light"><strong>Interface21</strong></span>, para enfocarse en Spring y sus módulos adicionales. Entre las versiones más relevantes, <span class="text-mb-blue-light"><strong>Spring 2.0</strong></span> (2006) introdujo mejoras como un soporte más avanzado para la configuración de beans y nuevas herramientas para simplificar el desarrollo.
    </p>

    <p>
        Para el año <span class="text-mb-blue-light"><strong>2007</strong></span> Interface21 pasó a llamarse <span class="text-mb-blue-light"><strong>SpringSource</strong></span> y lanzó Spring 2.5, lo que impulsó el desarrollo de aplicaciones web basadas en <span class="text-mb-blue-light"><strong>Model View Controller (MVC)</strong></span> con herramientas como <span class="text-mb-blue-light"><strong>Spring MVC</strong></span> y otros módulos adicionales​.
    </p>

    <p>
        En el <span class="text-mb-blue-light"><strong>2009</strong></span>, SpringSource fue adquirida por <span class="text-mb-blue-light"><strong>VMware</strong></span>, permitió que el framework se expandiera hacia soluciones en la nube.

        <span class="text-mb-blue-light"><strong>Spring Boot</strong></span> es lanzado en <span class="text-mb-blue-light"><strong>2014</strong></span> como una extensión simplificada del framework Spring. Este nuevo proyecto de Spring Boot nació con la idea de hacer aún más rápido el proceso de configuración y desarrollo de aplicaciones mediante <span class="text-mb-blue-light"><strong>configuraciones predefinidas</strong></span> y facilitar el despliegue de aplicaciones en contenedores y entornos en la nube.
    </p>
